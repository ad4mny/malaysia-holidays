# Contributing to malaysia-holiday

Thank you for your interest in contributing! Here are some guidelines:

## Development Setup

1. Fork and clone the repository
2. Install dependencies: `npm install`
3. Make your changes in the `src/` directory
4. Build the project: `npm run build`
5. Test your changes: `node example.js`

## Code Style

- Use TypeScript for all new code
- Follow existing code formatting
- Add JSDoc comments for public APIs
- Keep functions focused and simple

## Testing

Before submitting a PR:

- Ensure the project builds without errors: `npm run build`
- Test the CLI: `node dist/cli.js list`
- Test the API: `node example.js`

## Commit Messages

- Use clear, descriptive commit messages
- Start with a verb (Add, Fix, Update, etc.)
- Keep the first line under 50 characters

## Pull Requests

- Create a new branch for your feature/fix
- Update documentation if needed
- Add examples if adding new features
